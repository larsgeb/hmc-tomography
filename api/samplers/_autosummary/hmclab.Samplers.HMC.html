Sampling using the Hamiltonian Monte Carlo algorithm.

Parameters

  • samples_hdf5_filename (str) ‚Äì String containing the (path+)filename of the HDF5 file to contain the samples. A required parameter.

  • distribution (_AbstractDistribution) ‚Äì The distribution to sample. Should be an instance of a subclass of _AbstractDistribution. A required parameter.

  • stepsize (float) ‚Äì A positive float representing the time step to be used in solving Hamiltons equations. Has a strong influence on acceptance rate. An essential tuning parameter.

  • amount_of_steps (int) ‚Äì A positive integer representing the amount of integration steps to be used in solving Hamiltons equations. Has a medium influence on acceptance rate. An essential tuning parameter.

  • mass_matrix (_AbstractMassMatrix) ‚Äì An object representing the artificial masses assigned to each parameters. Needs to be a subtype of _AbstractMassMatrix. Has a strong influence on convergence rate. One passing None, defaults to the Unit mass matrix. An essential tuning parameter.

  • integrator (str) ‚Äì String containing ‚Äúlf‚Äù, ‚Äú3s‚Äù or ‚Äú4s‚Äù for a leapfrog, 3-stage or 4-stage symplectic integrator respectively, to be used during the trajectory calculation.

  • initial_model (_numpy) ‚Äì A NumPy column vector (shape dimensions √ó 1) containing the starting model of the Markov chain. This model will not be written out as a sample.

  • proposals (int) ‚Äì An integer representing the amount of proposals the algorithm should make.

  • online_thinning (int) ‚Äì An integer representing the degree of online thinning, i.e. the interval between storing samples.

  • randomize_stepsize (bool) ‚Äì A boolean enabling the randomization of the stepsize, which helps avoiding MCMC resonance.

  • diagnostic_mode (bool) ‚Äì A boolean describing if subroutines of sampling should be timed. Useful for finding slow parts of the algorithm. Will add overhead to each function.

  • ram_buffer_size (int) ‚Äì An integer representing how many samples should be kept in RAM before writing to storage.

  • overwrite_existing_file (bool) ‚Äì A boolean describing whether or not to silently overwrite existing files. Use with caution.

  • max_time (float) ‚Äì A float representing the maximum time in seconds that sampling is allowed to take before it is automatically terminated. The value None is used for unlimited time.

  • autotuning (bool) ‚Äì A boolean describing whether or not stepsize is automatically adjusted. Uses a diminishing adapting scheme to satisfy detailed balance.

  • target_acceptance_rate (float) ‚Äì A float between 0.0 and 1.0 that is the target acceptance rate of autotuning. The algorithm will try to achieve this acceptance rate.

  • learning_rate (float) ‚Äì A float larger than 0.5 but smaller than or equal to 1.0, describing how aggressively the stepsize is updated. Lower is more aggresive.

Raises

  • AssertionError ‚Äì For any unspecified invalid entry.

  • ValueError ‚Äì For any invalid value of algorithm settings.

  • TypeError ‚Äì For any invalid value of algorithm settings.
